# 计划跟踪号订单类型 与 逐 SKU 三件套就绪 — 验证记录

> **状态**: 数据层验证完成(2026-05-29),功能未实现。本文档记录已用**实时金蝶**逐条验证的事实,作为后续功能方案的依据。
> **验证方式**: `内销/金蝶实时查询` 的 `KingdeeClient` 直查生产环境(串行,SDK 连接池脆弱,未用并行)。
> **样本**: 呼吸管 `AS/AK2604007`、潜水镜 `AS/AK2602009`、泳镜(内销)`DS/DK262027S`、泳镜子单 `AS/AK2604001-1..4`、蛙鞋 `AS2603009`、样品 `AY26*`。

---

## 1. 计划跟踪号编号体系 (FMTONo)

前缀是**结构化编码**,不是固定的 "AK"。

### 第二个字母 = 订单类型
| 字母 | 类型 | 特征 | 产出 |
|---|---|---|---|
| **`S`** | 完整订单 | 对外发货给客户 | 07 成品 + 03 包装 |
| **`K`** | 备货单 / 半成品单 | 长交期件提前备 | 05 半成品 + 06 未包装 |
| **`Y`** | 样品单 | 小批量打样(5~180 件),客户含"样品室"/各国潜在客户 | 不计入生产就绪 |

### 第一个字母 = 业务线 / 客户组织
| 字母 | 业务线 | 格式示例 |
|---|---|---|
| **`A`** | 外销(MARES 等) | `AK2604007`(无尾缀) |
| **`D`** | 内销(刀刀 / 凯瑞克) | `DK251003S`(带尾缀字母 S,与类型 S 无关) |
| **`W`** | 瑞弧 | `WS2510004` |

> 子单后缀 `-1/-2/-3/-4` 真实存在(`AS2604001-1..4` ↔ `AK2604001-1..4` 逐子单配对)。QuickPulse `factory.py:142` 的 `LIKE '{mto}%'` 正好能聚合子单。

## 2. 最关键发现:出口与内销是两种制造模式

| | **出口(A 系列,MARES)** | **内销(D 系列,刀刀/凯瑞克)** |
|---|---|---|
| 模式 | **按单备货 (make-to-order)** | **自有品牌按库存 (make-to-stock)** |
| 完整单 S | 纯包装(只产 07 + 03) | 多为**自包含**(自做 05→07,常跳过 06 层) |
| 备货单 K | 按**客户 SKU** 下 06/05 生产订单 | 通用成品/库存池(按颜色款,内部 aux) |
| K↔S 号码配对 | **有意义**(同号=同产品两阶段) | **纯属巧合** |
| 06↔07 aux 桥 | ✓ 成立 | ✗ 不成立 |

**内销号码巧合实证**: `DS262027S` 卖凯瑞克 `NANOK9200AF` 泳镜;`DK262027S` 备刀刀 `GT73` 泳镜——两个产品、两个客户,只是恰好都叫 262027。`DS262027S` 的 BOM 只消耗 05+03(无 06),即 NANOK 泳镜 05→07 直做、自包含。

## 3. 逐 SKU "三件套" 就绪 — 三条桥接键

对完整订单 S 的每个成品 SKU,要回答:**产品(未包装)备好没 · SKU 专属包装备好没 · 通用包装备好没**。

### 桥 1 — 产品/未包装(06 ↔ 07):同 FAuxPropId,**仅出口成立**
- 算法:对每个 `(07码, aux)`,把 `07→06` 换码 + **同 aux**,**全局搜 `PRD_MO`** 找产这个 06 的单。
- 产在哪张单 = 供给的 K(可能 `AK{同号}`、可能别的单、可能 S 自己;若该 06 全局不存在 = 产品无 06 层,05→07 直做)。
- 出口的 06 生产订单带**客户 SKU aux**,与 07 成品 aux 是同一个 ID → 干净命中。
  - 呼吸管 `AS2604007`:6/6 命中(`07.04.076 aux=225990 需2550` ↔ `AK2604007 的 06.04.076 PRD_MO aux=225990 计划2552`,数量近 1:1)。
  - 潜水镜 `AS2602009`:4/4 命中。
- **内销失败 0/27**(06 用内部 aux)→ 内销改为"DS 自包含,就绪看其自产 05→07"。
- 就绪量:看该 06 MO 的 **`FStatus`**(`4`在产 / `5`完工 / `6`结案)**+ 按 `FMoBillNo` 追 `PRD_INSTOCK`**(**不要**用入库单自身 aux——入库后会变成内部潘通规格码)。

### 桥 2 — SKU 专属包装(贴纸/镜盒贴/彩盒):**子串包含,出口+内销都成立**
- 包装 aux 解析出的规格串(`BD_FLEXSITEMDETAILV.FF100001`)**内嵌客户 SKU 码**。
- 匹配规则 = **「07 成品的规格串」作为子串出现在「包装的规格串」里**。
  - 呼吸管:07 `701510/BKBK` ⊂ 贴纸 `产品条码贴纸9*4.5CM,…,701510/BKBK`(6/6)
  - 潜水镜:07 `8802 43` ⊂ `8802 43 / MS17AP 外箱贴纸…`
  - 内销:07 `NANOK9700AF-2` ⊂ `合格证贴纸…型号:NANOK9700AF-2…`
- **SKU 码格式因产品线而异(`701510/BKBK` vs `8802 43` vs `NANOK9700AF-2`),必须用子串包含、不能用固定正则。**

### 桥 3 — 通用包装(外箱/说明书/骨袋):订单级
- 挂在 S 自己名下采购,直接读 S 的 `PUR_PurchaseOrder` / 采购入库,不分 SKU、不跨单。
- 例:`AS2604007` 说明书 8520/8520 齐、产品贴纸 6 变体全齐、**PE骨袋 8520/0 未到**。

### 适用范围矩阵
| 桥 | 机制 | 出口 | 内销 |
|---|---|:--:|:--:|
| 1 产品/未包装 | 同 aux 全局搜 06 | ✓ | ✗(走 DS 自包含 05→07) |
| 2 SKU 专属包装 | 07 规格串 ⊂ 包装规格串 | ✓ | ✓ |
| 3 通用包装 | S 自采购,订单级 | ✓ | ✓ |

## 4. 对功能设计的硬约束
1. **先按首字母 A/D 分流**:出口走桥 1 找 K;内销走"DS 自包含 + 库存池"。
2. **找 06 一律按 `(06码 + aux)` 全局搜 `PRD_MO`,绝不靠号码配对**(内销号码巧合、出口 06 可能在别单)。
3. 就绪量用 06 生产单 `FStatus` + `FMoBillNo` 追入库,**不要用入库单 aux**。
4. 包装用**子串包含**匹配(SKU 码格式随产品线变化)。
5. **AY 样品单排除**在生产就绪之外。

## 5. 已被实测推翻并纠正的错误假设(记录以防重犯)
| 错误假设 | 实测反例 | 纠正 |
|---|---|---|
| S↔K 走 BOM/领料 物料流 | `AS2604007` BOM 只有 07+03、领料 0 行 | 无 BOM/领料/源单外键,只能靠 aux |
| aux 语言不互通、无法自动对 | 比错了表:应比 06 **生产订单** aux(225990),不是入库单 aux(内部码) | 06 生产订单 aux = 07 SKU aux |
| "S 产 05 ⟹ 无 K" | 潜水镜 `AS2602009` 自产 05 仍有 `AK2602009` | 按 (06码+aux) 全局搜,别从 05 推断 |
| 蛙鞋无 06 层 | 全局 83 个 MO 产 `06.05.04` | 06 可能产在号码不匹配的别单 |
| 出口机制可套用内销 | 内销 06↔07 aux 桥 0/27 | 必先按 A/D 分流 |
