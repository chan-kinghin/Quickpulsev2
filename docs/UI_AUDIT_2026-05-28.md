## 🐛 Bug / 功能性

### 1. 🔴 Dashboard 用户菜单缺少 Inventory 入口
- **位置**: `src/frontend/dashboard.html:44-54`
- **现象**: Dashboard 顶部"用户"下拉菜单只有 同步管理 / 使用分析 / 退出登录。**没有跳转到 `/inventory.html` 的入口**。Inventory 的菜单能跳回 Dashboard，但反向不行 —— 单向导航死路。用户进了 dashboard 必须手动改地址栏。
- **建议**: 在 dashboard.html 用户菜单里加 `<a href="/inventory.html">物料库存查询</a>`。

### 2. 🔴 同一份 `main.css` 在不同页面 cache-buster 不同步
- **位置**:
  - `dashboard.html:11,19` → `?v=20260512`
  - `inventory.html:8,11` → `?v=20260528`
- **现象**: 改了 `main.css` 后只 inventory 会刷新缓存，dashboard 还在吃 5/12 的旧版本。CSS 是共享的，但 cache-buster 没同步。
- **建议**: 用构建脚本注入统一版本号；或把版本号集中到一个 Jinja2 变量 / 服务端模板里。

### 3. 🟡 左下角 F11 提示位置写死成 400px，和 chat 实际宽度 384px 错位 16px
- **位置**: `dashboard.html:908-913` (`right-[400px]`), CSS `.chat-sidebar` width: 384px (`main.css:661`)
- **现象**: chat 打开时左下角 F11/ESC 提示距离右边 400px，chat sidebar 距离右边 384px —— 提示和 chat 之间有 16px 缝。是开发时随手 +16 留 buffer 吗？看起来更像写错。
- **建议**: 改成 `right-[392px]`（chat 宽度 + 8px gap），或 `right-[400px]` 但把 chat 宽度也 +16px 对齐。

### 4. 🟡 全屏 + Chat 同时打开会让表格被 chat 覆盖
- **位置**: `dashboard.html:60` (`max-w-full` when `isFullScreen`)、`dashboard.html:28` (`margin-right: 384px` when `chatOpen`)
- **现象**: 全屏时 main 拉到 100% 宽，但 chat 打开还会再 push 384px —— 两个 transform 互不知道。fullScreen + chat 同时开，表格右侧 384px 被覆盖。
- **建议**: 全屏切换时如果 chat 已开就先关闭，或全屏时禁用 chat push。

## 🎨 视觉 / 一致性

### 5. 🟡 Inventory 缺底部色彩图例
- **位置**: `inventory.html` 没有；`dashboard.html:781-797` 有
- **现象**: Inventory 候选列表用 5 种 ErpClass 颜色 badge（外购/自制/委外/虚拟件/成品），但页面底部没图例。用户得点进去看 `erp_class_label` 才知道颜色对应什么。dashboard 反而有完整图例。
- **建议**: Inventory 也加一行图例，或者直接把图例做成可复用组件。

### 6. 🟡 物料类型颜色在两页之间不完全一致
- **Dashboard**:（`main.css:42-64`）成品=violet, 自制=emerald, 包材=sky, 委外=amber
- **Inventory**:（`inventory.html:466-475`）ErpClass 1 外购=sky, 2 自制=emerald, 3 委外=amber, 4 虚拟件=slate, 9 成品=violet
- **现象**: Dashboard "包材"用 sky，Inventory "外购 (ErpClass 1)"用 sky —— 概念其实是物料 vs 物料族，不是 1:1 对应。视觉上同色容易混淆。
- **建议**: 抽一份 design tokens（`--erp-finished-color`、`--erp-purchased-color`...），两页共用一份；并在 README 里说明颜色 → 概念映射。

### 7. 🟡 照片入口用 📷 emoji，跟 lucide 图标体系不一致
- **位置**: `dashboard.html:177` (parent photo button)、`dashboard.html:810` (inline panel header)
- **现象**: 整个 app 用 `data-lucide="..."` SVG 图标，只有照片相关用 📷 emoji。emoji 在不同 OS 显示不一样，和 SVG 视觉密度也不一样。
- **建议**: 改用 lucide 的 `image` 或 `camera` 图标。

### 8. 🟡 筛选 chip 默认 4 个全亮，看不出"未筛选"
- **位置**: `dashboard.js:19` `materialTypes: { '成品': true, '自制': true, '包材': true, '委外': true }`
- **现象**: 4 个 chip 默认都 active，但功能上等于"不筛选"。用户视觉上看不出当前是"全选"还是"已筛选"状态。`filter-chip-active` 还各自带不同颜色，更像是"4 个都选了"。
- **建议**: 默认全部 inactive（视觉上灰），点了任一变 active 才生效；或加个"全部"chip，4 个细分 chip 默认 inactive。

### 9. 🟡 Dashboard 表头里 sortable `<th>` 大量重复，每列 ~17 行 markup
- **位置**: `dashboard.html:357-548` 之间 10 个列头
- **现象**: 每个 sortable 列头都是几乎逐字复制（仅 column key 不同）。改一个字段要同步 5 处（th + td + tfoot + columns[] + dashboard.js sort logic）。
- **建议**: 用 `<template x-for="col in sortableColumns">` 抽表头模板。**注意**：CLAUDE.md 警告过 `columns[]` 重构是高风险，因为 HTML 里有 `columns[N].visible` 索引引用 —— 要谨慎，最好 visible 改成 `colVisible(col.key)` 函数调用，再抽模板。

### 10. 🟢 Aux fallback 用的是 ⚠ 字符（非图标）
- **位置**: `dashboard.html:611`
- **现象**: 跟其他列的 lucide 图标风格不一致。但功能可用。
- **建议**: 可以改成 `data-lucide="alert-triangle"` w-3 h-3。优先级低。

## 🏗️ 布局 / 信息架构

### 11. 🟡 用户菜单是 hover dropdown，但里面都是顶层独立页面
- **位置**: `dashboard.html:39-54`、`inventory.html:35-52`
- **现象**: 同步管理、使用分析、Inventory 都是同级独立页面，藏在"用户"hover 里，多一次点击。新用户可能根本不知道有库存查询页。
- **建议**: 改成 horizontal nav tabs（MTO 查询 / 库存查询 / 同步 / 分析），右侧只保留 用户名 + 退出。这样 #1 的死路问题也顺带解决。

### 12. 🟢 MTO 搜索框宽 max-w-md，Inventory 搜索框宽 max-w-xl
- **位置**: `dashboard.html:63` vs `inventory.html:63`
- **现象**: 两页搜索区视觉宽度差挺多。MTO 号短（10 位），inventory 输入长。不算 bug，但一致性可提升。
- **建议**: 选一种统一，或者根据输入预期长度分别保留但加上"搜索区域"标准 padding/对齐。
