QuickPulse

MTO 查询 库存查询 预警 同步 分析

使用说明

QuickPulse 是「产品状态明细表」看板：输入一个 计划跟踪号（MTO，如 AK2510034）， 系统会从金蝶 K3Cloud ERP 把这张单子下面每个物料的 需求 / 入库 / 领料 / 发货 拼齐，让你一眼看清齐套进度。

下面先讲每个页面是干嘛的（业务向），再讲 系统背后那套金蝶取数逻辑（点开折叠块看）。

页面导览

MTO 查询 /dashboard

主力页。输入计划跟踪号，看这张单下每个物料的明细表： 需求数、已入库、已领料、已发货、齐套率、超领标红。

怎么读：齐套率接近 100% = 料齐；某行标红 = 超领（领的比申请的多）。

库存查询 /inventory

按物料名称 / 规格 / 编码模糊搜，查金蝶 即时库存（按仓库 / 批号 / 辅助属性）。

注意：这页是实时直连金蝶、不走缓存， 所以每次查询稍慢（1–5 秒），但数字是当下最新的。

预警 /alerts

两个标签：超领（实发 > 申请）和 超发（已发货 > 订单量）。点任意一行可下钻到该 MTO 明细。

口径：只看「正偏差」单边量，不是重复计算；样品单（Y 单）默认已排除。详见下方折叠块。

同步 /sync

把金蝶数据批量拉进本地缓存，并显示各数据表的 新鲜度（最后同步时间、是否滞后）。

什么时候用：缓存类报表（如预警）依赖它。看到「数据滞后」就来这页触发一次同步。

分析 /admin

按时间范围看接单 / 出货等汇总分析视图。

提示：偏管理视角的聚合数据，按需选时间段。

金蝶数据逻辑（点开看）

下面是系统从金蝶取数、对账时遵循的规则 —— 都是踩过坑后校正过的版本。看不懂可以跳过，不影响日常使用。

① 数据从哪来：BOM 优先 + 三源入库合并

查一个 MTO 时，系统的取数路径是：

PRD_MO (生产订单) → 拿到母件信息 + 单号
│ 按 FMOBillNO 关联
PRD_PPBOM (生产用料清单) → 这张单到底有哪些物料
│ 每个物料并行查三个入库来源，全部合并：
├─ PRD_INSTOCK (SCRKD02_SYS · 生产入库)
├─ STK_InStock (RKD01_SYS · 标准采购入库)
└─ STK_InStock (RKD03_SYS · 委外入库)
▼ 跨表把 FRealQty 求和
聚合 → 产品状态明细表

BOM 优先：以 PPBOM 作为「这张单有哪些物料」的唯一真相。 三源合并：同一个物料编码可能同时在三种入库单里有记录， 所以永远是把三处并集求和，绝不只看一处。

② 物料怎么分类（自制 / 外购 / 委外）

收料去哪张单，由物料主数据上的 BD_MATERIAL.FErpClsID 决定：

ErpClsID 类型 主要收料单 次要（也要查）
1外购 STK_InStock RKD01PRD_INSTOCK
2自制 PRD_INSTOCK SCRKD02RKD01 / RKD03
3委外 STK_InStock RKD03PRD_INSTOCK
4虚拟件 （无收料）—
9成品（福伦特自定义） PRD_INSTOCK SCRKD02RKD01

显示标签（主料 / 辅料 / 外销包材 / 半成品 / 委外加工）则另走一条线： 用 CategoryID.FName（经 PPBOM 的 FMaterialId.FCategoryId）—— 因为 ErpClsID 做标签不靠谱 （抽样里 22/23 都返回 "2"，连外箱、委外瓶盖都算 2）。

反例（别这么干）：① 按物料编码前缀（07/05/03…）路由 —— 在 02/03/06/08 类里错 30–67%； ② 按 PPBOM.FMaterialType 路由 —— 这租户里 99.94% 都是 1，毫无信息量。

③ 入库单类型速查（委外是 RKD03，不是 RKD02）
单据 单据类型号 含义
PRD_INSTOCKSCRKD02_SYS生产入库（自制）
STK_InStockRKD01_SYS标准采购入库（外购）
STK_InStockRKD03_SYS委外入库

易错点：委外入库在 RKD03_SYS， 不是 RKD02_SYS（RKD02 查出来是 0 行）。旧代码 / 旧文档多处搞错过，已校正。

同一物料可多源并存：一个物料编码在一个 MTO 内可能同时有 PRD_INSTOCK + RKD01 + RKD03 的记录 —— 始终并集，不要假设单一来源。

④ 计划跟踪号（MTO）前缀含义

以 AK2510034 为例，开头两个字母各有含义：

第 1 个字母 = 业务线
A = 外销（MARES 等）
D = 内销（刀刀 / 凯瑞克）
第 2 个字母 = 订单类型
S = 完整订单
K = 备货 / 半成品单
Y = 样品单

出口（A）= 按单备货：AK 与 AS 通过「号码 + SKU 辅助属性」桥接成立。 内销（D）= 自有品牌按库存：DS 自包含，号码配对纯属巧合，不要硬桥。

⑤ 单据状态码（A 排除，B/C/D 计入）
状态码 含义 是否计入查询
A创建（草稿）排除
B已审核计入
C已确认计入
D重新审核计入

收料类查询一律过滤 FDocumentStatus IN ('B','C','D') —— 不能只取 ('C','D')，否则会漏掉「已审核未确认」的单。

⑥ 缓存 vs 实时：两条数据路径
实时（Live）

直连金蝶 API，1–5 秒，数字最新。MTO 查询默认走实时，库存查询只走实时。

缓存（Cache）

本地 SQLite 快照，<100ms，需先「同步」。预警等批量比对类报表走缓存。

缓存可能滞后，所以 /health 同时是存活探针 和新鲜度闸门（同步超过 24 小时会判定不健康）。关键决策前先到「同步」页确认新鲜度。

⑦ 预警口径（超领 / 超发怎么算）

超领：在 mto + 物料 粒度上，实发 > 申请 的 单边正偏差之和 —— 不是重复计算。 注意申请量常只填象征值，所以个别倍数会偏大；页面同时给出「净领料偏差」（含少领），整体往往是少领。

超发：mto + 物料 粗粒度比对（已发货 vs 订单量，已排除已关闭订单）。 含辅助属性差异时需人工复核。

两条诚实规则：① 数量字段缺失（NULL）的物料被跳过、不当 0 处理，需人工核对； ② 样品单（订单类型 Y）默认排除，不算超领 / 超发。
